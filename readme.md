# 五子棋游戏设计文档 (更新版)

## 1. 概述

本文档旨在详细描述一个基于 Python 和 Pygame 开发的五子棋游戏。该游戏具备图形化的用户界面、支持本地双人和在线联机两种模式，并通过多线程技术优化了网络对战的用户体验。

## 2. 核心功能

*   **图形化主菜单**: 游戏启动后，用户可以通过点击UI按钮来选择游戏模式，而非通过命令行。
    *   **本地游戏 (Local Game)**: 在同一台电脑上进行双人对战。
    *   **创建在线游戏 (Create Online Game)**: 作为主机（服务器）等待另一名玩家加入。
    *   **加入在线游戏 (Join Online Game)**: 作为客户端连接到主机。
*   **标准五子棋规则**: 在15x15的棋盘上，任意一方（黑棋或白棋）先在横、竖、斜任意方向上连成五子即获胜。
*   **清晰的游戏界面**:
    *   实时绘制棋盘和棋子。
    *   在屏幕顶部中央显示**回合指示器** (Turn Indicator)，明确提示当前应由哪方落子（例如 "Your Turn", "Opponent's Turn"）。
    *   游戏结束后，在屏幕中央显示胜利或失败信息。
    *   游戏结束后，提供一个**返回主菜单**的按钮，方便用户重新开始新的一局。
*   **流畅的在线体验**:
    *   采用**多线程**处理网络通信，解决了在等待对方落子时UI界面卡顿的问题，保证了界面的实时响应。
    *   当玩家在对方回合尝试落子时，会收到 "Not your turn" 的临时文字提示。
*   **精确的用户输入**: 仅响应鼠标**左键**点击，避免了右键或滚轮滚动导致的误操作。

## 3. 技术架构

项目采用模块化设计，将游戏逻辑、UI渲染、网络通信和配置项分离，以实现高内聚、低耦合。

*   **技术选型**:
    *   **核心语言**: Python 3
    *   **图形与交互**: `Pygame` 库
    *   **网络通信**: `socket` (Python标准库)
    *   **并发处理**: `threading` 和 `queue` (Python标准库)

*   **项目结构**:
    ```
    gobang/
    ├── main.py         # 程序主入口，包含游戏主循环和模式选择逻辑
    ├── game_logic.py   # 封装纯粹的游戏核心逻辑（棋盘、规则、胜负判断）
    ├── ui.py           # 负责所有图形界面的绘制和UI事件处理
    ├── network.py      # 封装了TCP网络通信，统一处理服务器和客户端角色
    └── config.py       # 存放所有硬编码的常量，如颜色、尺寸、字体大小等
    ```

## 4. 核心组件详解

### 4.1. `game_logic.py` - 游戏逻辑模块

*   **`Game` 类**: 管理整个游戏的状态。
    *   `board`: 一个15x15的二维列表，`0`表示空，`1`为黑棋，`2`为白棋。
    *   `current_player`: 记录当前应落子的一方。
    *   `check_win(row, col)`: 核心胜负判断算法。每次落子后，从该位置出发，检查水平、垂直和两条对角线四个方向上是否有连续五个同色棋子。
    *   `run_move(row, col)`: 执行一步完整的走棋流程，包括验证有效性、落子、检查胜负和切换玩家。

### 4.2. `ui.py` - 图形界面模块

*   **`UI` 类**: 负责所有渲染任务和UI交互。
    *   `draw_board()`: 绘制棋盘网格。
    *   `draw_pieces(board)`: 根据 `Game` 类的 `board` 状态绘制所有棋子。
    *   `draw_menu()`: 绘制包含三个模式选择按钮的主菜单。
    *   `draw_turn_indicator(text)`: 在屏幕顶部绘制回合状态文本。
    *   `draw_return_button()`: 在游戏结束后绘制返回主菜单的按钮。
    *   `pixel_to_grid(pixel_x, pixel_y)`: 将Pygame窗口的像素坐标转换为棋盘的网格坐标。
    *   `get_menu_choice(pos)` / `get_return_button_click(pos)`: 判断鼠标点击位置是否在特定按钮的矩形区域内。

### 4.3. `network.py` - 网络通信模块

*   **`Network` 类**: 一个统一的网络接口，既能作为服务器，也能作为客户端。
    *   `start_server()`: 绑定IP和端口，启动一个TCP套接字并监听，等待客户端连接。
    *   `connect_to_server(host)`: 创建一个客户端套接字并连接到指定的主机IP。
    *   `send_move(move)`: 使用 `json` 序列化走棋坐标（一个元组），并通过套接字发送。
    *   `receive_move()`: 阻塞式接收数据，直到收到对方的走棋数据或连接断开的信号，然后用 `json` 解码。

### 4.4. `main.py` - 主程序模块

*   **`main_menu(screen, ui)`**: 游戏启动后的第一个循环，负责绘制和处理主菜单的交互。根据用户的按钮点击，决定进入哪个游戏循环。
*   **`local_game_loop(screen, ui)`**: 本地对战模式的游戏循环。在一个循环内处理两个玩家的输入和游戏状态更新。
*   **`online_game_loop(screen, ui, is_host)`**: 在线对战模式的游戏循环。这是最复杂的部分：
    1.  **初始化**: 根据 `is_host` 参数决定是启动服务器还是连接服务器。
    2.  **启动后台线程**: 创建并启动一个 `network_receive_worker` 线程。该线程专门负责调用 `net.receive_move()` 这个阻塞操作，避免主UI线程被冻结。
    3.  **线程间通信**: `network_receive_worker` 线程将接收到的走棋数据放入一个 `queue.Queue` 中。
    4.  **主线程轮询**: `online_game_loop` 在其主循环中，以**非阻塞**的方式 (`queue.get_nowait()`) 检查队列中是否有新的走棋数据。
    5.  **UI与逻辑分离**: 主线程只负责处理用户输入、渲染UI和从队列中取数据来更新游戏状态，从而保证了界面的流畅。

## 5. 游戏流程

1.  **启动**: 运行 `main.py`，进入 `main_menu` 循环，显示主菜单。
2.  **模式选择**: 用户用鼠标左键点击一个按钮。
    *   **选择 "Local Game"**:
        *   进入 `local_game_loop`。
        *   玩家轮流点击棋盘落子。
        *   游戏结束后，显示胜利信息和返回按钮。点击按钮后，返回主菜单。
    *   **选择 "Create Online Game"**:
        *   进入 `online_game_loop` (is_host=True)。
        *   启动服务器，在控制台打印本机IP，UI显示 "Waiting for player..."。
        *   启动后台接收线程。
        *   等待客户端连接后，游戏开始。主机执黑先行。
    *   **选择 "Join Online Game"**:
        *   进入 `online_game_loop` (is_host=False)。
        *   提示用户在控制台输入主机IP。
        *   连接成功后，启动后台接收线程，游戏开始。客户端执白后行。
3.  **在线对弈**:
    *   轮到当前玩家时，其客户端响应鼠标点击，下棋，并通过网络发送坐标。
    *   对方客户端的后台线程接收到坐标，放入队列。
    *   对方客户端的主循环从队列中取出坐标，更新本地游戏状态和UI。
    *   双方交替进行，直到分出胜负或有一方断开连接。
4.  **结束**: 游戏结束后，显示结果和返回按钮，点击后关闭网络连接并返回主菜单。
